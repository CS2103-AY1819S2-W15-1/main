PROJECT: Fashion Match

Overview

Fashion Match is a desktop wardrobe management application used as a guide for your fashion necessities. The user interacts with it using a CLI, and it has a GUI created with JavaFX. It is morphed from Address Book over a period of 8 weeks. It is written in Java, and has about 10 kLoC.

Calculating statistics : stats or statistics

Gives an overview of the various statistics of the apparels/wardrobe.
Format: stats or statistics

Typing stats or statistics will give you the following several information about your wardrobe:

  • Your favorite apparel

  • Your favorite color

  • Total number of apparel in wardrobe

  • Total number of different colored apparels

  • Your least favorite apparel (a reminder for you to wear them more often!)

  • How clean or dirty is your wardrobe

Statistics feature

Show statistics of the apparels of the current wardrobe. ==== Current Implementation Below is a high level visualisation of the statistics command and a step-by-step explanation of an example scenario for the usage of stats.

Figure 1. High Level Visualisation of Statistics Command

  1. User enters command stats. The command is received by FashionMatchParser, which then creates a StatisticsCommandParser object.

  2. StatisticsCommandParser will then create a relevant Statistic object.

  3. After that, a StatisticsCommand object is created with the Statistic object as parameter and return to LogicManager.

  4. LogicManager then calls StatisticsCommand.execute(), which calls Model.getFavApparel(), Model.getFavColor(), Model.getFilteredApparelList().size(), Model.getTotalColor, Model.getLeastFavApparel and Model.getCleanOrDirty.

  5. The result will be shown on the UI

Design Considerations

  • Alternative 1 (current choice): Go through all the apparels in Fashion Match and displays statistics whenever the statistics command is called.

    • Pros: This approach is less intensive on the computational power of the user’s PC, as the command is only executed whenever the user calls it.

    • Cons: Less user friendly as statistics will not be real-time

  • Alternate 2: Updates statistics in real-time and whenever the statistics command is used.

    • Pros: This approach is more user-friendly as users would not need to type stats command after new apparel is changed.

    • Cons: Time-consuming and not scalable. This method could be more computationally more intensive, especially if the user has a large wardrobe.

Proposed additions for v2.0

Statistics command will be a real-time feature that captures the state of the user’s wardrobe and apparel in real-time and does not need to call the Statistics command each time. It will also occupy a dedicated space in the UI.
