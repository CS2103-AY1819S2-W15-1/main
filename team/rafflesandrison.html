PROJECT: Fashion Match

Overview

Fashion Match is a command line application that is morphed from an addressbook application. This is a Software Engineering project that required us to use their code as a base and we decided to morph it into an wardrobe management system. This application enables normal users to: Manage the arrangement of apparels in the wardrobe Wash and wear specific apparel Recommend outfit combination Usage statistics

The part of Fashion Match that I implemented is the Wardrobe Management Feature which comprises of list [OPTION], sort [OPTION], and swap [INDEX_X] [INDEX_Y]. Also, I handled the display of the apparel details on the User Interface when an apparel is selected. More details about these features will be elaborated below including documentation of it in the user and developer guide.

Sort Apparel feature

Sort the list of items based on the options given. The sorting will be increasing order of the name of the option specified. Simulating an event in the real world where users arrange their wardrobe according to the category of their choice. For example, in a wardrobe where the folded shirts and pants are stacked on top of one another, the user might want to put all the shirts on top of all the pants.

Current Implementation

The sort command takes in one parameter. The valid arguments are: NAME, COLOR, and TYPE.

  • Name : The full name of the apparel.

  • Color : The color of the apparel.

  • Type : The clothing type of the apparel.

e.g sort color will sort the apparels by color. If there are only 3 colors - Red, Black, Blue - in the wardrobe, calling sort color will arrange all the apparels in the following color starting from top - Black, Blue, Red.

Figure 1. High Level Visualisation of Sort Command

When the user inputs the sort command in the CLI, the input is parsed, and the sorting option specified are validated.

sort keyword will call a constructor for SortCommandParser to parse the sort option, color, creating a SortCommand for LogicManager.

LogicManager executes SortCommand which calls Model to sort the apparels in the list by color, and commits the changes to Storage.
